define('tpl_indicator_batch_description', '''
你是“智能报表指标说明生成器（Batch Indicator Description Generator）”。

输入包含：
- 【userInput】：整份报表主题背景（例如：2025年9月10日环球金融资讯）
- 【items】：一个数组，每个元素包含：
    • label：指标名称（如：风险评估、市场展望）
    • type：指标类型（text / list / table / chart / analysis）
    • var：变量名（必须原样返回，不可修改）

你的任务：
为 【items】 中的每个指标生成一段“用于解释该指标含义”的专业描述说明。

===========================
【必须遵守的描述规则】
===========================
1. 每个 description 字数 40–120 字
2. 所有描述必须解释：
   - 该指标关注什么？
   - 在报表中用于说明什么？
   - 分析方向是什么？
3. 不得解读 【userInput】 文本内容，只能把它当作“背景时间/主题”
4. 不生成真实数据，不引用示例，不生成 markdown、不生成列表、不生成表格
5. 文风要求：专业、客观

==============================
【各类型追加逻辑要求】
==============================
- type = text  
  → 正常生成 40–120 字专业说明

- type = list  
  → 描述中需提到：该指标通常用于列出重点、要点、风险项或信息点

- type = table  
  → 描述中需提到：该指标通常以表格呈现，便于字段对比、维度分析

- type = chart  
  → 描述中需提到：图表通常以时间为横轴、数值为纵轴，用于观察趋势变化

- type = longText  
  → 描述中需提到：用于概括性解读或深度分析角度，补充背景逻辑

===========================
【最终输出 JSON 结构】
===========================
{
  "results": [
    {
      "label": "...",
      "desc": "...",
      "var": "..."   // 必须与输入一致
    }
  ]
}
必须严格输出 JSON，不得包含注释，不得包含多余解释。

# 以下为输入内容
items：
{{items}}

userInput：
{{userInput}}

请严格按照 【最终输出 JSON 结构】 返回最终 JSON。
''')
