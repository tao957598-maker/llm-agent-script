define('tpl_md_content', '''
你是 Apache POI-TL 模板设计专家与 Markdown 模板工程师。

输入是一份「报告结构 Schema」，你必须基于它生成 **Markdown 模板内容**，
其中所有 POI-TL 模板标签必须使用 [[ ]] 占位符。

【输入 Schema】
{{n_md_schema}}

--------------------------------------------
必须严格遵守以下规则（包含强制“双换行”排版规范）
--------------------------------------------
1. 所有 POI-TL 标签一律使用 [[ ]] 占位符：
   - 文本变量：         [[var]]
   - 表格变量：         [[$tableVar]]
   - 图表变量：         [[@chartVar]]
   - 列表循环块：       [[?varList]] ... [[!varList]]

2. 输出必须是纯 Markdown 文本：
   - 不允许出现 JSON
   - 不允许出现 markdown 代码块 ```
   - 不允许出现解释或说明
   - 只输出最终 Markdown 模板内容

3. **所有需要换行的地方必须使用“双换行”（两个换行符 \n\n），包括：**
   - section 标题与内容之间
   - section 与下一个 section 之间
   - 文本变量块之间
   - 表格前后必须双换行
   - 图表前后必须双换行
   - 列表块前后必须双换行
   - 表格中每个结构块之间必须双换行
   - 图表单独一行上下均为双换行
   - mixed 类型中不同 block 必须双换行分隔

--------------------------------------------
1. 顶层标题生成规则
--------------------------------------------
- 第一行必须输出：
  # [[reportTitle]]

- reportTitle 必须使用变量形式输出：
  - 必须是 [[reportTitle]]
  - 不允许写成 reportTitle
  - 不允许替换为实际文本
- "#" 与变量之间必须有空格

**此行后必须紧跟两个换行符。**

--------------------------------------------
2. 章节生成规则
--------------------------------------------
每个 section 生成：

## section.title

（必须双换行）

如果 section.desc 不为空，则单独一行输出 desc 文本，然后继续双换行。

不同 section 之间必须使用“双换行 + 双换行”（即空出一个完整空行块）。

--------------------------------------------
3. 文本变量块：blockType = "text"
--------------------------------------------
输出格式如下：

labelText：[[varName]]

**此块结束后必须双换行。**

--------------------------------------------
4. 表格变量块：blockType = "table"
--------------------------------------------
格式如下：

① 表格标题行：  
tableTitle(标题行不能有[[$tableVar]])

（双换行）

② Markdown 表格结构：

| header1[[$tableVar]] | header2 | header3 | ... |
| ------ | ------ | ------ | --- |
| [colVar1] | [colVar2] | [colVar3] | ... |

规则说明：
- tableTitle 完全原样输出，不插入任何变量（不加 [[$tableVar]]）。
- Markdown 表格的第一列表头（header1）后必须自动追加 [[$tableVar]]。
- 列名：| [colVar1] | [colVar2] | [colVar3] | ... | 必须使用单[]包含，觉得不能使用[[]]包含
- 其他 header 不追加任何变量。
- 表格结构前后必须空两行（双换行）。
- 表格结束后也必须空两行（双换行）。

**表格整体前后必须双换行，结束后必须再追加双换行。**


--------------------------------------------
5. 图表变量块：blockType = "chart"
--------------------------------------------

所有 chart 块必须严格按照如下格式输出（每个 chart 独立一块）：
【强制格式：每个图表固定 4 行结构】
第 1 行：chartTitle 文本（只输出纯标题文本，不做任何字符串替换）
第 2 行：[[@chartVar]] （变量必须独占一行，不能缺失，不能并入标题）
第 3 行：空行（必须）
第 4 行：空行（必须）

也就是说，每一个图表块的最终 Markdown 模板必须长这样：
chartTitle(标题行不能有[[@chartVar]])
[[@chartVar]]

（中间是两个换行符，即一行空行）

示例（仅示意，不要照抄文案）：
美国10年期孳息率
[[@us10YYield]]

日本10年期孳息率
[[@jp10YYield]]

英国10年期孳息率
[[@uk10YYield]]

内地10年期孳息率
[[@cn10YYield]]

强制约束（禁止以下行为）：
- 不允许省略第 2 行的 [[@chartVar]]
- 不允许把 [[@chartVar]] 合并到标题行里
- 不允许删除 [[@chartVar]] 或只保留标题
- 不允许用表格形式展示图表（禁止 | 标题A | 标题B | 这种表格）
- 不允许横向并排多个图表，所有图表必须逐个纵向排列

无论一个 section 中有多少个图表，必须按上述「标题一行 + 变量一行 + 双换行」的模式依次输出

6. 列表变量块：blockType = "list"

列表块必须使用如下格式，并且 **每一行之间必须强制加入一个空行（两个换行符）**：

[[?varList]]

[[itemVar1]]

[[itemVar2]]

...

[[!varList]]

严格要求：
- 列表块顶部必须有一个空行（在 [[?varList]] 前）
- 每个 [[itemVarX]] 前必须自动插入一个空行
- [[!varList]] 前必须自动插入一个空行
- 不允许移除空行、合并空行、或自行压缩空行
- 不允许使用 markdown 项目符号（如 "-" 或 "*"）

列表块上下均必须 **双换行**。

--------------------------------------------
7. 排版要求（已强化成双换行规范）
--------------------------------------------
- 章节之间必须两个换行符
- 文本块之间双换行
- 表格上下双换行
- 图表上下双换行
- 列表上下双换行
- 所有 block 渲染结束后还需补一个双换行
- 不得出现单换行导致视觉粘连

--------------------------------------------
8. 输出要求
--------------------------------------------
- 必须输出纯 Markdown 内容
- 必须严格按照 Schema 中的 sections 顺序生成
- 不得输出任何 JSON 或解释文字

--------------------------------------------

请基于 {{n_md_schema}} 输出最终 Markdown 模板内容，并确保所有需要换行的位置都使用“双换行”。
''')


// Step 4：生成 Markdown 内容

//## Step 5: 替换 n_md_content 中的占位符：[[xx]]-->{{xx}}
n_format_md_content_result,act_api.doExecute?name='report_format_md_data'&params='{{n_md_data}}'
when('n_format_md_content_result.code==200',{
  //转换成功,将最新内容赋值给n_md_content
  assign('n_md_content', '''{{n_format_md_content_result.data}}''')
})
// Step 6：生成最终文件名（若用户没传 fileName 就自动生成）
n_md_file_info, act_func?script='''
    import java.text.SimpleDateFormat
    def tempName = templateName+".md"
    //生成到tmp目录下（会被定时清理）
    def baseDir ="/appdata/template"
    // 兼容 Windows / Linux 路径简单处理
    def sep = baseDir.contains("\\\\") ? "\\\\" : "/"
    def fullPath = baseDir.endsWith(sep) ? (baseDir + tempName) : (baseDir + sep + tempName)

    def info = [:]
    info.put("dir", baseDir)
    info.put("fileName", tempName)
    info.put("mdFilePath", fullPath)
    return info
'''

log('MD file path: {{n_md_file_info.fullPath}}')

// Step 7：写入 Markdown 文件
n_write_result, act_func.write?filename="{{n_md_file_info.mdFilePath}}"&data="{{n_md_content}}"

// Step 8：返回结果
assign('result', '''
Markdown 模板已生成并保存：
- 报告名称：{{templateName}}
- 保存路径：{{n_md_file_info.mdFilePath}}
''')

// Step 9：Md转word(复用report_word_to_pdf)

  '''

  //n_kbs,act_kbs.fileSyncKbSpace?filePath='system/tmp/api/hello_test_report_template_20260112101109.docx'

      def wordMap=n_store_template_word_path.data[0]

      def wordFileId = wordMap.get(wordFileName)  
      def result=[:]
      result.put("code",200)
      result.put("wordFileId",wordFileId)
      return result
    }
'''
  log('{{n_main_result}}')
})
