# 页面操作流程生成系统总结

## 1. 系统概述

本系统是一个基于用户自然语言需求，自动生成页面操作流程的智能系统。通过分析用户需求，匹配相关页面，并生成详细的操作步骤序列，帮助用户快速完成目标任务。

## 2. 核心文件结构

| 文件名 | 功能描述 |
|-------|----------|
| `agt_gen_steps_main.txt` | 主程序入口，协调整个流程 |
| `agt_get_list_url.txt` | 根据自然语言选择最佳匹配的页面 |
| `agt_get_start_end_url.txt` | 确定起始和结束URL，以及途经URL |
| `agt_search_page_graph.txt` | 搜索页面图中的最短路径 |

## 3. 流程详解

### 3.1 主流程入口

**输入参数：**
- `system_name`：系统名称（必填）
- `userInput`：用户需求（必填）

**流程步骤：**
1. 设置参数，将用户需求等信息转换为JSON格式
2. 调用`agt_get_list_url`获取与用户需求匹配的页面列表
3. 检查是否获取到页面列表：
   - 若未获取到，返回空数组
   - 若获取到，继续执行后续步骤
4. 调用`agt_get_start_end_url`获取起始URL、结束URL和途经URL列表
5. 检查起始URL和结束URL是否相同：
   - 若相同，直接查询该页面的收集数据
   - 若不同，调用图数据库搜索最短路径
6. 生成页面操作信息，返回最终结果

### 3.2 页面匹配流程（agt_get_list_url）

**功能**：根据用户自然语言需求，从数据库中匹配最相关的页面路由。

**流程步骤：**
1. 执行SQL查询，获取指定系统的所有页面数据（包含当前URI和内容）
2. 将页面数据按每10个元素分组
3. 对每组数据，调用模板分析匹配度，生成匹配结果
4. 汇总所有匹配结果
5. 根据用户需求和匹配结果，返回最匹配的页面路由

**输出格式：**
```json
[
  {"url1": "序号. 步骤描述"},
  {"url2": "序号. 步骤描述"},
  {"url3": "序号. 步骤描述"}
]
```

### 3.3 起始/结束URL确定流程（agt_get_start_end_url）

**功能**：确定用户操作的起始URL、结束URL和途经URL列表。

**流程步骤：**
1. 从匹配的页面列表中提取第一个页面作为当前URI
2. 执行SQL查询，获取该页面的详细信息
3. 检查查询结果：
   - 若无数据，返回无数据状态
   - 若有数据，继续执行
4. 检查当前URI和访客重定向URI是否相同：
   - 若相同，无需权限控制，起始URL为当前URI
   - 若不同，需要权限控制，起始URL为访客重定向URI
5. 调用模板生成包含起始URL、结束URL和途经URL的结果

**输出格式：**
```json
{
  "status": true,
  "start_url": "对应start_url",
  "end_url": "对应end_url",
  "passThrough": ["url1", "url2", "url3"]
}
```

### 3.4 页面图搜索流程（agt_search_page_graph）

**功能**：在页面关系图中搜索从起始URL到结束URL的最短路径，支持途经指定节点。

**流程步骤：**
1. 检查是否提供了系统名称，若未提供则设为空字符串
2. 检查是否有途经节点（passThrough）：
   - 若有途经节点，构建包含途经节点的Cypher查询
   - 若无途经节点，构建普通的最短路径查询
3. 执行Cypher查询，搜索最短路径
4. 返回路径节点和关系信息

**输出**：包含节点列表、关系列表和路径长度的结构化数据

## 4. 数据流转

1. **用户输入** → `agt_gen_steps_main` → 接收系统名称和用户需求
2. **`agt_gen_steps_main`** → `agt_get_list_url` → 传递系统名称和用户需求
3. **`agt_get_list_url`** → 数据库 → 查询页面数据
4. **`agt_get_list_url`** → `agt_gen_steps_main` → 返回匹配的页面列表
5. **`agt_gen_steps_main`** → `agt_get_start_end_url` → 传递系统名称和页面列表
6. **`agt_get_start_end_url`** → 数据库 → 查询页面详细信息
7. **`agt_get_start_end_url`** → `agt_gen_steps_main` → 返回起始URL、结束URL和途经URL
8. **`agt_gen_steps_main`** → 数据库/图数据库 → 查询页面数据或搜索路径
9. **`agt_gen_steps_main`** → 模板渲染 → 生成操作步骤信息
10. **`agt_gen_steps_main`** → **用户** → 返回最终操作流程

## 5. 最终输出格式

系统最终生成的操作流程信息采用JSON格式，包含以下字段：

- **No**：顺序数字
- **Current_URI**：当前页面URI
- **Operation_Sequence**：操作序列，包含序号、步骤描述和操作类型target id
- **Target_URI**：完成操作序列后导航到的页面路径

**示例输出：**
```json
[
  {
    "No": 1,
    "Current_URI": "/login",
    "Operation_Sequence": [
      "1. 输入用户名: admin",
      "2. 输入密码: password",
      "3. 点击`登录`按钮 [login-button]"
    ],
    "Target_URI": "/dashboard"
  },
  {
    "No": 2,
    "Current_URI": "/dashboard",
    "Operation_Sequence": [
      "1. 点击`交易`菜单 [transaction-menu]",
      "2. 点击`转账`选项 [transfer-option]"
    ],
    "Target_URI": "/transfer"
  }
]
```

## 6. 技术特点

1. **自然语言处理**：能够理解用户的自然语言需求，匹配相关页面
2. **图数据库集成**：利用图数据库搜索页面之间的最短路径
3. **模板引擎**：使用模板渲染生成结构化的操作步骤信息
4. **SQL查询**：从关系型数据库中获取页面数据
5. **模块化设计**：将功能拆分为多个独立模块，便于维护和扩展

## 7. 应用场景

1. **自动化测试**：生成测试用例的操作步骤
2. **用户指南**：为用户生成详细的操作指南
3. **流程优化**：分析和优化用户操作流程
4. **系统集成**：作为其他系统的流程生成组件

## 9. 系统优化建议

1. **缓存机制**：对频繁查询的页面数据和路径结果进行缓存，提高响应速度
2. **错误处理**：增强错误处理机制，提供更友好的错误提示
3. **日志记录**：添加详细的日志记录，便于问题排查和系统监控
4. **性能优化**：优化SQL和Cypher查询，减少查询时间
5. **可扩展性**：设计插件机制，支持自定义操作步骤生成逻辑

## 10. 总结

本系统通过整合自然语言处理、数据库查询和图数据库搜索等技术，实现了从用户自然语言需求到详细操作步骤的自动生成。系统设计模块化，流程清晰，能够有效地帮助用户理解和执行复杂的页面操作流程。

系统的核心价值在于：
- 减少用户学习成本，提供直观的操作指南
- 标准化操作流程，提高操作一致性和准确性
- 自动化生成操作步骤，节省人工编写成本
- 支持复杂系统的流程导航，提升用户体验

通过不断优化和扩展，该系统可以应用于更多场景，为用户提供更加智能和个性化的操作引导服务。
