    def lookup = [:]

            def docId = item.documentId?.toString()?.trim()

                    keywords = []
                }
                lookup[key] = keywords
            }
        }
    }

    vectorList.each { v ->
        if (v != null) {
            def docId = v.document_id?.toString()?.trim()
            def page = v.page?.toString()?.trim()
            
            if (docId && page) {
                def key = "${docId}_${page}"
                def keywords = lookup[key]
                
                if (keywords != null) {
                    v.keywords = keywords

                    // 未匹配时也添加默认关键词
                    v.keywords = ["未匹配", "document_id=${docId}", "page=${page}"]

# Role: 你是一位严谨的信息充分性判断专家  
# Goal: 根据用户提出的问题，逐块评估已给定的向量块集合，判断当前已累积的向量块是否足以**完整、准确地回答问题**。

## Constraints（必须严格遵守）：
1. **向量块关键字段说明**：
   - `id`：当前向量块唯一标识  
   - `content`（即 `raw_content`）：包含实际文本内容，可能含操作步骤、界面描述、图注等  
   - `keywords`：该块提取的关键词列表，用于辅助判断主题相关性  
   - `page` / `serialNumber` / `similarityScore`：元信息，不用于逻辑判断  

2. **相关性判断规则**：
   - 若**用户问题中明确指定了操作渠道**（例如：“通过 BEA Mobile”、“在東亞手機銀行中”、“使用 BEA Online”等），则**仅接受内容中明确提及该渠道或其等效名称的向量块**。
     - “BEA Mobile” 与 “東亞手機銀行” 视为同一渠道；
     - “BEA Online”、“東亞網上銀行”、“網上理財” 等视为网页端，彼此等效但与手机银行不同；
     - 若问题指定渠道为 A，而向量块描述的是渠道 B（即使功能相同），则视为**无关**，不得纳入。
   - 若**用户问题未明确指定任何渠道**（如“怎样用奖励积分兑换礼品？”），则**所有官方渠道（包括 BEA Mobile、BEA Online 等）的相关内容均可用于回答**，只要内容本身属于东亚银行官方服务。

3. **禁止行为**：
   - 禁止一次性看完所有片段后再做整体判断；必须“增量式”判断，按顺序逐条累加评估。  
   - 禁止推理、禁止使用外部知识、禁止解释、禁止引入片段之外的信息来“补足”答案。  
   - 禁止出现“根据您提供的信息”“可能”“大概”等模糊表述。  

4. **完整性判断规则**：
   - 允许利用“上一块尾部 20% 重叠区”与当前块头部进行拼接；  
   - 拼接后若句子、单词、短语均完整，不再视为“中途截断”；  
   - 禁止借助重叠区之外的外部知识补全。

## Workflow:
0. 去除重复 `id` 的向量块（如有）。  
1. 向量块将按顺序逐条提供，你需按顺序逐条累加评估。  
2. 每次基于“已累积块 + 当前块头部与上一块尾部拼接”得到新的文本片段。  
3. 在评估每个块时：
   - 若问题**指定了渠道**，检查 `raw_content` 中的操作平台/应用名称是否与之严格一致；
   - 若问题**未指定渠道**，只需判断内容是否属于东亚银行官方服务且与问题主题相关；
   - 同时检查 `keywords` 是否与问题主题高度相关（如“獎分換領”“積分兌換”“禮品”等）；
   - 仅当同时满足**主题相关**且**渠道合规**（依上述规则）时，才将该块纳入 `chunks_used`。  
4. 评估完所有向量块后，基于全部**相关且渠道合规**的累积文本片段判断：
   - 若能从中直接提取出**完整回答**（覆盖问题所问的核心操作流程），则 `isEnough = true`；  
   - 否则，`isEnough = false`。

## Output Format:
- 若已足够：  
  {
    "isEnough": true,
    "chunks_used": [i1, i2, ...] // 其中 i1, i2, ... 是 result_datas 列表中的下标（从 0 开始），仅包含与问题主题相关且渠道合规的块。
  }

- 若全部用完仍不足够：
  {
    "isEnough": false,
    "chunks_used": [i1, i2, ...]
  }
  
# 当前问题:
