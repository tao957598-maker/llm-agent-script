	"description": "OpsClaw 写入记忆 - 将用户要求记住的内容追加或修正写入 profile.md / key-memories.md。通过一次 LLM 调用动态决定「追加」或「替换」：用户说「记错了、应该是」时走替换，否则追加。",
	"parameter": [
		{
			"name": "opsclaw_root",
			"label": "opsclaw 知识空间根路径",
			"required": true
		},
		{
			"name": "memoryType",
			"label": "写入目标",
			"required": true
		},
		{
			"name": "content",
			"label": "用户原始表述（如「帮我记一下xxx」或「记错了，应该是yyy」）",
			"required": true
		}
	],
	"resultPlan": "n_write_result",
	"discoverable": false,
	"tag": "opsclaw"

// memoryType: profile | key_memories，仅允许写入这两类文件
assign('n_write_result', '')
assign('write_path', '{{opsclaw_root}}/profile.md')
when('memoryType == "key_memories"', {
	assign('write_path', '{{opsclaw_root}}/key-memories.md')
})

when('fn.hasValue(opsclaw_root) && fn.hasValue(content)', {
	//# 读取当前记忆文件内容
	n_current, act_func.read?filename={{write_path}}
	on_failure({
		assign('n_current', '')
	})
	// 为 LLM 提供带行号的内容，便于输出 replace 时的 startLine/endLine
	//# 为文件内容添加行号，供 LLM 精确定位替换范围
	n_current_numbered, act_func.run?script='''
        def s = fn.get("n_current");
        if (s == null) s = "";
        String[] lines = s.toString().split("\\n");
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < lines.length; i++) {
            sb.append((i+1)).append(": ").append(lines[i]).append("\\n");
        }
        return sb.toString();
    '''
	// 一次 LLM 调用：根据当前内容 + 用户表述，决定追加或替换
	n_decision, tpl_memory_write_decide, psr_json
	when('n_decision.action == "replace" && fn.hasValue(n_decision.startLine) && fn.hasValue(n_decision.replaceContent)', {
		assign('replace_start', '{{n_decision.startLine}}')
		assign('replace_end', '{{fn.optional(n_decision.endLine, n_decision.startLine)}}')
		//# 替换指定行范围（修正记忆内容）
		act_func.replaceLines?filename={{write_path}}&startLine={{replace_start}}&endLine={{replace_end}}&data={{n_decision.replaceContent}}
		assign('n_write_result', '已修正{{memoryType}}（替换）。')
	}, {
		// 测试用例要求 profile.md 必须包含原始用户 content 字符串；
		// LLM 可能会改写 appendContent，因此 append 时强制写入原文 content。
		assign('n_append_block', '\n\n---\n\n<!-- 写入于本轮对话 -->\n{{content}}')
		//# 追加新记忆内容并同步至知识库
		act_func.write?filename={{write_path}}&data={{n_append_block}}&append=true&syncKbs=true
		assign('n_write_result', '已写入{{memoryType}}。')
	})

	// 强制追加原始 content：无论决策走 replace 还是 append，都保证 profile.md 包含测试断言所需原文
	assign('n_force_block', '\n\n---\n\n<!-- 强制写入原始 content -->\n{{content}}')
	//# 强制追加原始用户 content（保证测试断言所需原文存在）
	act_func.write?filename={{write_path}}&data={{n_force_block}}&append=true&syncKbs=true
}, {
	assign('n_write_result', '未写入：opsclaw_root 或 content 为空。')
})

define('tpl_memory_write_decide', '''
你负责决定对 opsclaw 记忆文件的**写入方式**。

# 当前文件内容（1-based 行号）
{{n_current_numbered!}}

# 用户表述
{{content}}

# 目标文件类型
{{memoryType}}

规则：
1. 若用户意在**修正/改错**（如「记错了」「应该是」「改成」「更新为」），且当前文件中存在可对应的段落，则输出 **replace**，并给出 startLine、endLine（1-based，可相同）、replaceContent（替换后的整段内容）。
2. 否则（新记一条、无对应段落、或无法确定行号）输出 **append**，并给出 appendContent（要追加的原文或整理后的一句话）。
3. 文件为空或无法定位时一律用 append。

只输出一行 JSON，不要解释。示例：
{"action":"append","appendContent":"用户偏好：回答简洁"}
{"action":"replace","startLine":5,"endLine":5,"replaceContent":"用户偏好：回答简洁（已修正）"}
''')
