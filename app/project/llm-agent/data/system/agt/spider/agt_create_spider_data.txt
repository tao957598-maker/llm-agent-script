【任务目标】
根据输入的steps列表（包含steps_id、python_code、start_uri、end_uri、steps等信息），生成一个新的list，每一条代表一个“完整可跳转步骤”。

约束：
1、根据步骤列表+python代码+start_uri与end_uri总结生成出新的list
    每一条数据都代表一个可跳转的步骤，这个步骤可能是由原先的list中多个数据组合起来的
    其中代码会被压缩成这样的格式，表示一个会触发页面跳转的完整步骤：
    [{"type":"click","target":"id_xxxx"},{"type":"click","target":"id_xxxx"}]
2、run_steps后期会用来进行md5，因此中间尽量不要使用空格或其他无意义的符号
3、action_type只允许为："open_url":打开链接, "execute":执行操作
4、run_steps 中的type只允许为："open_url"、"input"、"click"、""四种，如果不在这四种内，则不参与生成代码
    当为"open_url"时格式为：{"type":"open_url","value":"xxx uri"}
    当为"input"时格式为：{"type":"input","target":"id_xxxx","value":"xxxx"}
    当为"click"时格式为：{"type":"click","target":"id_xxxx"}
    当为"select"时格式为：{"type":"select","target":"id_xxxx","option":"xxxx"}
5、每组数据必须存在start_uri、end_uri,start_uri为跳转前的uri，end_uri为页面跳转后的uri,当action_type为open_url的时候，start_uri=""
6、生成的数据必须为原先的输入生成的，不允许生成虚拟数据
7、生成的list每一条都必须是代表可以跳转的操作，如果不能跳转则忽略这个数据

1、分析start_uri、end_uri, 当认为这组数据会发生且只发生一次页面跳转的时候,将这组数据组合
