## ✅ 改进设计方案：分为两层

### 一、通用层（所有文档都适用）

| 节点类型 | Label      | 主要字段都是字符串类型                                                            | 说明            |
| ---- | ---------- | ----------------------------------------------------------------- | ------------- |
| 文件   | `Document` | `document_id`, `document_name`, `document_type`, `upload_date`, `source_path`,`page_count`,`summary` | 唯一标识文件  |
| 页面   | `Page`     | `document_id`, `keywords`, `page_number`, `page_title`, `summary`       | 页级信息（可抽取关键句）  |
| 章节   | `Section`  | `section_id`, `section_title`, `level`, `section_path`, `text`    | 逻辑结构划分（标题、层级） |

### 关系（通用层）

| 关系类型                    | 起点 → 终点              | 含义        |
| ----------------------- | -------------------- | --------- |
| `HAS_PAGE`              | `Document → Page`    | 文件包含页面    |
| `HAS_SECTION`           | `Document → Section` | 文件包含章节    |
| `SECTION_CHILD`         | `Section → Section`  | 上下级章节结构   |
| `PAGE_CONTAINS_SECTION` | `Page → Section`     | 页面内容映射到章节 |

---

### 二、领域层（可根据文件类型扩展）

| 节点类型  | Label       | 主要字段                                                     | 示例（报文类文件）                              |
| ----- | ----------- | -------------------------------------------------------- | -------------------------------------- |
| 实体    | `Entity`    | `entity_id`, `entity_name`, `entity_type`, `description` | “交易请求”、“授权响应”等                         |
| 概念    | `Concept`   | `concept_id`, `concept_name`, `definition`, `category`   | “余额查询”、“报文字段”                          |
| 属性项   | `Attribute` | `attr_id`, `attr_name`, `attr_value_type`, `attr_desc`   | “Field 37: Retrieval Reference Number” |
| 规则/约束 | `Rule`      | `rule_id`, `rule_name`, `rule_logic`, `rule_type`        | “响应码必须存在”                              |

### 关系（领域层）

| 关系类型            | 起点 → 终点              | 含义        |
| --------------- | -------------------- | --------- |
| `MENTIONS`      | `Section → Concept`  | 某章节提到某概念  |
| `DESCRIBES`     | `Section → Entity`   | 某章节描述实体   |
| `HAS_ATTRIBUTE` | `Entity → Attribute` | 实体拥有属性    |
| `HAS_RULE`      | `Entity → Rule`      | 实体关联约束或逻辑 |
| `RELATES_TO`    | `Concept ↔ Concept`  | 概念之间的语义关联 |
