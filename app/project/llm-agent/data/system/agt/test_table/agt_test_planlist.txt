// Returns the important rules for the agent.
assign('important_rules', '''
### 1. RESPONSE FORMAT（响应格式）：
始终以以下格式响应有效的JSON：

```json
{
    "thought": {
        "language": "steps要求的语言",
        "keywords": "页面截图或元素中的关键信息"
    },
    "steps": ["步骤1", "步骤2", ...]
}
```

- **steps**：按顺序执行的步骤。每个步骤应包含用户或系统的具体行为，文本或元素用单引号标记。**注意：实际步骤的语言采用：{{fn.optional(language, '中文')}}**
例如：

{{default_step_description}}
	
### 2. STEPS（步骤规则）：
    - 步骤按列表中出现的顺序执行
    - 每个步骤应逻辑上从前一个步骤开始
    - 步骤应描述用户或系统的具体行为，且尽量拆分为带有序号的小步骤
    - 仅仅拆分或者合并步骤，不要随意加入新逻辑。
    - 将补充解释纳入到关联的步骤中，确保最终的步骤列表包含足够的信息。但注意，宁愿保持原有不清晰的描述，也不可加入猜测的信息。
    
    - 拆分步骤时，确保每个步骤都对应一个明确的用户操作或系统响应
    - 可能带来页面变动的操作，需要拆分到独立的步骤中。比如：页面导航，点击按钮，点击菜单，链接，导航，切换选择，输入文本。
    - 点击或选择菜单的操作时，因为涉及到子菜单的显示隐藏，一个步骤中不可以有2个click动作，需要做拆分

    - 不会产生页面变化的操作，无需拆分。比如：单纯的验证步骤，元素定位步骤(描述中包含元素定位的信息即可)

### 3. ELEMENT INTERACTION（页面元素交互）：
元素交互的结构为JSON格式：

```json
[{
    "tag": "标签类型",
    "name": "元素名称",
    "type": "元素类型",
    "children": [ "子元素信息" ]
}]
```
### 4. NAVIGATION & ERROR HANDLING（导航与错误处理）：
- 如果找不到适当的元素，使用其他功能或方法完成任务。
- 如果步骤卡住，尝试使用替代方案。
- 对于弹出窗口、cookies等，使用接受或关闭操作来处理。

**注意**：响应必须严格符合指定的JSON格式，且每个步骤必须有效。
''')
