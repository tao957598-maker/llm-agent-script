define('tpl_page_summary_api', '''
# API / HTTP specification extraction
Analyze the following document slice. Treat it as **API contract** material (REST/OpenAPI/Swagger style): paths, methods, parameters, headers, auth, status codes, request/response bodies, error models.

## Output format (must match downstream pipeline)
Return a **JSON array** of objects (list of maps). Output text in **{{lang_output_text}}**. Same schema as the Web page summary agent:
```json
[{
  "page": 1,
  "requirement_type": "Functional",
  "section": "API group name",
  "title": "Endpoint or operation title",
  "description": "What the API does and key rules",
  "business_rules": ["rule1"],
  "acceptance_criteria": ["HTTP 200 with expected schema", "401 when token missing"],
  "dependencies": [],
  "testability": true
}]
```

## requirement_type
- Prefer **Functional** for endpoints, schemas, and error contracts.
- Use **Compliance** only if the text is purely legal/regulatory without callable API.
- Use **Background** only for glossary with no testable API surface.
- Use **Other** sparingly.

## Rules
1) Ignore pure TOC / navigation-only lines with no API semantics.
2) Do **not** merge Web UI concerns; focus on machine-readable contracts.
3) Split distinct operations into separate array entries when evidence supports it.
4) If the slice has no API-relevant content, return [].

## Downstream uses same JSON keys as Web — fill them with **API meaning** (not Web UI wording)
| Field | API-oriented content (English) |
|-------|--------------------------------|
| section | OpenAPI **tag** / resource group, or service name + area |
| title | Prefer **HTTP_METHOD path** (e.g. `POST /v1/{resource}`) or **operationId** if named in spec |
| description | One paragraph: purpose, auth (Bearer/API-key), major query/body params, idempotency if stated |
| business_rules | Constraints: required headers, rate limits, enum values, schema refs, deprecation |
| acceptance_criteria | Verifiable lines: expected **HTTP status**(es), error code mapping, response schema highlights |
| page | Keep document page/slice number from payload for traceability |

# Page payload
{{page_content}}

''')
