在diff格式中，减号(-)表示要删除的行，加号(+)表示要添加的行。
优化后的提示词：

## Unified Diff 内容生成规范

### 核心要求
1. **精准修改**：只输出实际发生代码变更的 hunk，移除未修改的上下文行
2. **保持行号**：严格基于原始代码行号进行定位和修改
3. **无额外内容**：不添加注释、假设或解释性文字

### 生成规则
- **范围精确**：每个 hunk 必须使用最小必要的行号范围（`@@ -起始行,行数 +起始行,行数 @@`）
- **拆分原则**：当多个独立修改点相距较远时，拆分为多个 hunk
- **内容精简**：删除所有 ` ` 和 ` ` 的未修改行标记

### 输出格式
1.说明diff输出格式
2.返回diff数据:
```diff
@@ -原起始行,原行数 +新起始行,新行数 @@
 需要删除的原始行（以 - 开头）
+需要添加的新行（以 + 开头）
```

### 工作流程
1. **分析需求**：理解当前代码功能与目标需求
2. **定位修改**：确定必须修改的具体代码行
3. **最小变更**：仅修改必要的代码行，保持其余部分不变
4. **必须解释输出格式**：说明diff输出格式
5. **格式验证**：确保输出符合标准的 Unified Diff 格式
6. **内容过滤**：移除所有未实际修改的上下文行

### 禁止事项
- 添加注释或说明性文字
- 包含未修改的代码行作为上下文
- 使用假设性修改
- 改变原始行号对应关系

### 示例
**错误示例**（包含未修改行）：
```diff
@@ -10,7 +10,7 @@
   line8
   line9
- old_line10
+ new_line10
   line11
   line12
```

**正确示例**：
```diff

+ new_line10
```
