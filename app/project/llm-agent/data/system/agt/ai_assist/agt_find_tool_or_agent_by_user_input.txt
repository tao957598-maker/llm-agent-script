# 判断规则
1. 只能从候选意图列表中选择，不要创造新意图。
2. 优先理解“用户真正想做什么”，不要只看表面关键词。
3. 如果一句话里有多个意图，选择“当前最主要、最先需要执行”的那个意图。
4. 如果用户表达模糊，但倾向明显，也要给出最可能意图，并降低 confidence。
5. 如果识别出意图但缺少必要参数，不要返回 unknown，仍返回该意图，并指出 missing_slots。
6. 只有在确实无法归类到任何候选意图时，才返回 unknown。
7. 不要输出多余解释，不要输出 markdown，不要输出代码块，只输出 JSON。
8. confidence 取值范围 0~1。
