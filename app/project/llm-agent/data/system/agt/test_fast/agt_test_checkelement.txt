// Use selenium
assign('pythonType', 'selenium', 'subfolder', '{{test_no}}', 'sys_hint_agt', '')

// Collect the new page information
act_rpa.send?command=CollectPageInfo

// Analyse page elements
tpl_task_element_info, psr_json?filter=id&format='id_\\d+'

// Generate python code according to decision
tpl_generate_check_script, psr_json_python, act_python.writeTmp?pythonCode={{sys_last_result}}
	
// Run selenium script through RPA client
n_python_result, act_rpa.send?name={{python_file}}

// Ouput the python script result
n_python_file, act_python.write?requirement={{current_step}}

// Write execution result
act_kb.merge?name=kb_testcase_step&columnValues={{fn.json('result,n_python_result.screenshot_filename', true)}}&where={{fn.json('test_no,test_no,step,current_step')}}

// 生成验证脚本

		 "llmType":"alias.middelqulity"
	 }*/
	你是熟悉selenium 4的从事银行业务的高级测试，需要校验页面数据。请结合页面信息生成"测试步骤"中用于校验的python函数。接着，请review该python函数并返回最终的JSON格式的review result。
	# python函数为：run(browser)
	它将被主模块用module.run(browser)的方式调用，请确保生成的代码符合测试脚本要求。

	# review该python函数的JSON格式：
	1. 代码中的元素不是来自"实际页面元素",需要替换。
	用JSON格式返回：{{fn.task(false).thought('假设的元素XXX，或者需要替换的元素XXX').suggest('建议修正方法XXXX')}}
	2. 代码中的元素都是来自"实际页面元素",不用替换。
	用JSON格式返回：{{fn.task().plan('next')}}。
	
	# 测试步骤为：
	{{current_step}}
	
	# 测试步骤涉及的实际页面元素(结构为json,[{tag:标签类型, name:元素名称,	type:元素类型, id: 用于定位的元素id, children:[子元素信息]}...])：
	{{tpl_task_element_info}}
	
	# 完整的页面元素
	{{CollectPageInfo}}
	
	{{sys_hint}}
	
	# 测试脚本要求：
	1. 只可以使用“实际页面元素"中列出的元素，不要自己假设不存在的元素
	2. “实际页面元素”中的id是加密后的id, 因此, 不可使用find_element_by_id
	3. “实际页面元素”没有提供xpath的信息, 可以使用autotest中的getXpathByItem获取到xpath后，再进行selenium 4的代码调用, 比如：browser.find_element(By.XPATH, 'xxx'), find_element(By.TAG_NAME, 'xxx')
	4. 务必谨慎处理硬编码金额的期望值, 这些值在未来发生变化时可能会导致用例失败
	

	# 其中来自autotest的可调用函数（通过from autotest import getXpathByItem, waitElement导入下面的函数）：
	其中item来自实际的页面元素,json的string类型，格式为：'{"name":"元素名称", "id": "用于定位的元素id"}'
	1. getXpathByItem(browser, item),基于item获取页面元素的xpath.参数browser是已经构建好的webdriver.Chrome实例,参数item来自实际的页面元素.
	2. clickElementByItem(browser, item, waitSecond, disableLog), 用于元素点击. 基于item查询对应的xpath,并基于xpath定位元素,进行点击.waitSecond为可选参数,默认等待1秒,对于搜索按钮建议等待5秒.disableLog为可选参数,默认为False,不禁止输出日志.点击成功返回True,否则返回False.
	3. waitElement(browser, item), 检查并等待item所指向的元素出现
''')

	{{current_step}}

''')
