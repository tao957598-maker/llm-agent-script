// Step1: 查询所有已配置的动态工具集及其内部工具
n_existing_toolset_result, act_api.doExecuteDynamic?name=tool_details_list&dynamicParamData='{"toolStatus":"1","mcpStatus":"1","statusLogic":"or"}'

assign("n_existing_toolset", "{{n_existing_toolset_result.data}}")

// 提取已有动态工具集中已包含的工具 code 列表
n_existing_tool_codes, act_func.run?script='''
    def toolsets = n_existing_toolset ?: []
    def codes = []
    if (toolsets instanceof List) {
        for (int i = 0; i < toolsets.size(); i++) {
            def ts = toolsets.get(i)
            def list = []
            if (ts.toolList instanceof List) {
                list = ts.toolList
            } else if (ts.tools instanceof List) {
                list = ts.tools
            }
            if (list instanceof List) {
                for (int j = 0; j < list.size(); j++) {
                    def t = list.get(j)
                    def code = t.code ? t.code.toString() : ""
                    if (code && !codes.contains(code)) {
                        codes.add(code)
                    }
                }
            }
        }
    }
    return codes
'''

// Step2: 向量搜索候选工具（无论是否有新版意图，都先搜索）
n_search_result, act_api.search?size=100&query="{{userInput}}"&toolType="{{toolType}}"&excludeToolType="{{excludeToolType}}"

// 格式化搜索结果：保证 n_tool_details 始终为工具对象列表
n_tool_details, act_func.run?script='''
    def raw = n_search_result ?: []
    if (raw instanceof Map) {
        if (raw.data instanceof List) return raw.data
        return []
    }
    if (raw instanceof List) return raw
    return []
'''

// Step3: 过滤掉已有动态工具集中已存在的工具，仅保留本次需要的新工具
n_new_tool_candidates, act_func.run?script='''
    def candidates = n_tool_details ?: []
    def existingCodes = n_existing_tool_codes ?: []
    def newTools = []
    if (candidates instanceof List) {
        for (int i = 0; i < candidates.size(); i++) {
            def t = candidates.get(i)
            def code = t.code ? t.code.toString() : ""
            if (code && !existingCodes.contains(code)) {
                newTools.add(t)
            }
        }
    }
    return newTools
'''

// Step4: 语义识别用户输入是否包含"新版"意图
// 使用 LLM 进行语义分析，判断用户是否想要使用新工具/新版本
n_intent_result, tpl_check_new_intent

// 解析 LLM 返回的意图结果
assign('n_has_new_intent','{{n_intent_result.hasNewIntent}}')

// Step5: 根据是否有新版意图进行分流
when('n_has_new_intent == "true"', {
    // 有新版意图：直接用新工具候选创建工具集
    when('fn.hasValue(n_new_tool_candidates)', {
        // 用过滤后的新工具覆盖 n_tool_details，确保只包含新工具
        assign('n_tool_details', '{{n_new_tool_candidates}}')

        // 交由 LLM 生成新工具集
        n_tools, tpl_choose_tools

        when('fn.hasValue(n_tools.toolList)', {
            // 选到了工具，创建新工具集
            assign('n_should_save', 'true')
            assign('n_tools_for_save', '{{n_tools}}')
        }, {
            // 选不到工具，回退到匹配已有新版工具集
            n_has_new_match, tpl_check_new_toolset_match
            when("fn.equals(n_has_new_match.matched, 'true')", {
                assign('n_tool_set_code', "{{n_has_new_match.toolsetCode}}")
            }, {
                assign('n_tool_set_code', '')
            })
        })
    }, {
        // 无新工具候选，回退到匹配已有新版工具集
        n_has_new_match, tpl_check_new_toolset_match
        when("fn.equals(n_has_new_match.matched, 'true')", {
            assign('n_tool_set_code', "{{n_has_new_match.toolsetCode}}")
        }, {
            assign('n_tool_set_code', '')
        })
    })
}, {
    // 无新版意图：先尝试匹配已有工具集，如果匹配不到，再用新工具创建
    n_has_existing_match, tpl_check_existing_match

    when("fn.equals(n_has_existing_match.matched, 'true')", {
        // 已有工具集匹配成功，直接使用
        assign('n_tool_set_code', "{{n_has_existing_match.toolsetCode}}")
    }, {
        // 已有工具集匹配失败，尝试用新工具创建
        when('fn.hasValue(n_new_tool_candidates)', {
            assign('n_tool_details', '{{n_new_tool_candidates}}')

            n_tools, tpl_choose_tools

            when('fn.hasValue(n_tools.toolList)', {
                assign('n_should_save', 'true')
                assign('n_tools_for_save', '{{n_tools}}')
            }, {
                assign('n_tool_set_code', '')
            })
        }, {
            assign('n_tool_set_code', '')
        })
    })
})

// Step6: 需要保存时，调用保存接口
when('n_should_save == "true"', {
    n_save_result, act_api.doExecuteDynamic?name=tool_set_save&dynamicParamData={{n_tools_for_save}}
    when('n_save_result.code == 200 || n_save_result.code == 50003', {
        assign('n_tool_set_code', "{{n_tools_for_save.toolsetCode}}")
    }, {
        assign('n_tool_set_code', '')
    })
})

// 最后返回结果
assign('n_result', '{{n_tool_set_code}}')

// 提示词定义：语义识别用户是否有"新版"意图
define('tpl_check_new_intent', '''
你是意图分析专家。请根据用户的输入，语义分析用户是否想要使用"新工具"、"新版工具"或"新版本"。

## 判断标准
用户明确表达以下意图时，视为"有新版意图"：
1. 明确说要"使用新工具"、"用新版"、"新版"、"最新版本"、"新版本"
2. 明确说要"创建新工具集"、"新建工具集"、"新增工具集"
3. 明确说"不想要旧的"、"不要老版本"、"换新版"、"升级"
4. 用户输入中包含上述意图的同义表达，如"最新工具"、"新的工具"等

用户只是在描述业务需求（如"帮我查订单"、"创建一个会议"）时，视为"无新版意图"。

## 输出格式（严格 JSON）
{
    "hasNewIntent": true/false,    // 是否想要使用新工具/新版
    "reason": "简要理由"            // 判断理由
}

## 用户输入：
{{userInput}}

请直接输出 JSON，不要有任何额外解释。
''')

// 提示词定义：判断现有工具集是否已有匹配
define('tpl_check_existing_match', '''
你是工具集匹配专家。根据用户输入，判断系统中已有的工具集列表中，是否已有工具集的工具与用户需求匹配。

## 判断标准
如果已有工具集的列表中，存在至少一个工具集的 description 或 name 与用户输入高度相关（精确或相关匹配），则视为"已匹配"。

## 输出格式（严格 JSON）
{
    "matched": true/false,        // 是否有现成的匹配工具集
    "reason": "简要理由",         // 为什么匹配或不匹配
    "toolsetCode": "xxx",         // matched=true 时必填，该工具集的唯一标识
    "toolsetName": "xxx",         // matched=true 时必填
    "toolList": [...]             // matched=true 时必填，该工具集的全部工具列表
}

## 用户输入：
{{userInput}}

## 已有工具集列表：
{{n_existing_toolset}}
(每个工具只包含 name/code/type/description 四个字段，toolList 中返回的工具必须原样复制这些字段的值)

请直接输出 JSON，不要有任何额外解释。
''')

// 提示词定义：从新工具候选中选取本次所需工具生成新工具集
define('tpl_choose_tools', '''
- Role: 智能办公系统管家
- Background: 当前向量搜索已返回一批与用户需求相关的工具，其中部分工具可能已被已有动态工具集覆盖。你当前看到的工具列表 n_tool_details 已过滤掉已有动态工具集中的工具，仅保留本次真正需要的新工具。请基于用户输入，从这批新工具中判断当前大模型是否真的需要调用它们，并选出最合适的工具生成新的动态工具集。
- Requirements:
1. 核心任务： 分析用户输入，从 n_tool_details 中判断是否真正需要调用这些新工具。如果已有动态工具集已经能满足需求，则不要创建新工具集（返回空 toolList）。
2. 字段规则：
   toolsetName：必须根据用户输入的核心意图生成一个简短、贴切的集合名称。例如，用户输入是"查询订单"，则可设为"订单查询工具"；是"会议创建"，则可设为"会议创建工具集"。
   toolsetCode：必须根据用户输入的核心意图生成一个简短、贴切的集合断码，英文下划线分割。例如，用户输入是"查询订单"，则可设为"order_query_tool_set"；是"会议创建"，则可设为"meeting_create_tool_set"。
   toolList：最多返回20个工具，按score从高到低排序。每个工具必须包含name, code, type, score这四个字段，其中description字段若源数据有则填入，没有则留空字符串。
   关键约束：toolList中所有返回的工具对象，其name、code、type和score字段的值必须严格、完整、原样复制自n_tool_details列表中对应工具的原始值。不允许自行修改、截断、重新表述或生成新值。description字段若源数据中有则原样复制，没有则填空字符串。
3. 匹配逻辑与评分 (score):
  精确匹配 (score: 9-10): 用户输入与工具的name或description高度相关。例如，用户问"查订单"，优先匹配agent_mcp_http_server02#getOrderInfoById。
  相关匹配 (score: 5-8): 用户需求与工具的description有部分关键词重叠或语义相关。例如，用户说"分析一下舆论"，可匹配agt_public_opinion_analysis_demo。
  模糊匹配 (score: 1-4): 需求与工具功能仅有微弱关联，或匹配到泛用类别（如OA）。例如，用户说"批个流程"，可能匹配到agt_my_test.txt.oa。
  无匹配 (score: 0): 工具集中无任何工具符合需求。此时toolList中返回一个name和description为空的对象，score为0。
4. 只返回score >= 5的数据集合；如果没有工具达到此阈值，返回空 toolList。
5. 如果判断已有动态工具集能够满足当前用户需求，即使 n_tool_details 中有工具，也不要纳入新工具集，直接返回空 toolList。
# 返回示例：
{
    "toolsetName": "xxxxx",
    "toolsetCode": "xxxxx",
    "description":"xxxxx",
    "toolList": [
        {
            "name": "",
            "code": "",
            "type": "",
            "description": ""
        }
    ]
}

# 用户输入：
{{userInput}}

# 工具列表（向量搜索结果，已过滤已有动态工具集中的工具）：
{{n_tool_details}}
(此列表是一个JSON数组，包含工具的name、code、type等原始字段（description可能有也可能没有）。toolList中返回的工具信息必须100%来源于此列表，需根据name和type匹配对应的工具，description有则复制，没有则填空字符串)

# 最终指令:
请严格遵循以上所有要求，分析用户输入，并从工具列表中匹配工具。始终优先考虑选择的agent。
请特别注意，toolList中每个工具的name、code、type、description字段值，必须是n_tool_details列表中对应字段的完整、原文复制。description字段若有则复制，没有则填空字符串。
当没有工具达到 score >= 5 时，返回空的 toolList，不要返回一个假的工具对象。
返回的toolsetName、toolsetCode以及 description 在任何情况下都不能为空，并且description要有具体的意义，不能是空字符串，如果userInput中涉及"新版","新工具"等字样，需要在新生成的toolsetCode中加上 _new后缀，在description中也要体现出来这个是新的版本的工具集描述。
将您的分析结果以指定的JSON格式返回。

''')

// 提示词定义：有新版意图但选不到工具时，匹配已有新版工具集
define('tpl_check_new_toolset_match', '''
你是工具集匹配专家。用户表达了"使用新工具"或"新版"的意图，但在向量搜索的新工具候选中没有找到合适的工具。

现在请从已有的工具集列表中，查找是否有名为"新版"、"new"、"v2"等关键词的动态工具集，或者工具集中包含新版/新版本工具的工具集。

## 判断标准
1. 工具集名称或描述中包含"新版"、"new"、"v2"、"新版本"、"最新"等关键词
2. 工具集中包含的工具是较新版本的工具（如带有 _new、_v2、_latest 等后缀的工具）
3. 工具集的 description 中提到"新版本"、"新版"等字样

## 输出格式（严格 JSON）
{
    "matched": true/false,        // 是否有匹配的已有新版工具集
    "reason": "简要理由",         // 为什么匹配或不匹配
    "toolsetCode": "xxx",         // matched=true 时必填，该工具集的唯一标识
    "toolsetName": "xxx",         // matched=true 时必填
    "toolList": [...]             // matched=true 时必填，该工具集的全部工具列表
}

## 用户输入：
{{userInput}}

## 已有工具集列表：
{{n_existing_toolset}}
(每个工具只包含 name/code/type/description 四个字段，toolList 中返回的工具必须原样复制这些字段的值)

请直接输出 JSON，不要有任何额外解释。
''')
