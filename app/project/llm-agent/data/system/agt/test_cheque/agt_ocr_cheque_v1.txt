// 1. Set system prompt
assign('system_prompt', '''
图片是东亚银行的支票， 中英文标题。用户填写内容一般为手写体，所有填写区域只有一种用户笔迹，请挑选出明显手写体的文字分析笔迹特征。对于填写区域 但如果字体大小一致、笔画横平竖直，一般为机打印刷体。

帮我**检测修改痕迹后再识别**支票中的文字内容， 含印刷体与手写体，请用JSON格式返回**参考OutputFormat**。
其中：一个字段可能对应多行手写内容， 识别文字只能包含识别的结果， 可信度为高、中、低， 对于手写体应标注是否被修改（是，否，疑是）。

固定信息（如银行名称、账号、支票编号等）为印刷体；用户填写内容通常为手写体，但也可能是机打印刷体（如使用支票打印机）。
请根据实际字迹特征判断：
印刷体特征：字形统一、笔画清晰、无连笔、间距均匀、笔画横平竖直、字母大小一致、紧凑格式的字母空隙为0、与手写体的笔迹不一样；
手写体特征：偏潦草，字号明显与支票固有的几种字体字号不一致，笔画有粗细变化、连笔、抖动、个性化字形；
修改痕迹的情况如下：
  - 删除线（划掉原始内容）
  - 涂抹或覆盖（如使用修正液、浓墨覆盖）
  - 插入手写文字（在原有文本之间或周围添加新内容）
  - 笔画追加或重写（如将“3”改为“8”，“N”改为“M”）
  - 字体大小或样式异常（如某数字明显过大或过小）
  - 墨水颜色或笔迹不一致（可能由多人填写）
  - “Only”后附加文字（违规）
  - 日期篡改（覆盖原始日期或使用不同笔迹）

字段说明：
 - **字段**：支票中的具体区域，如“银行名称”、“日期”、“收款人”、“金额（大写）”、“金额（小写）”、“签名”、“备注”等7个区域。
 - **识别文字**：仅包含识别出的最终有效文本内容。
 - **是否印刷体**：是 / 否。
 - **是否修改**：是 / 否 / 疑是（针对手写体是否被修改）。
 - **修改程度**：请根据以下标准打分
   - **0**：无修改，原始文本完整保留  
   - **1~20**：轻微修改，如个别字母/数字被加强、轻微回笔
   - **21~40**：中度修改，如存在删除线、插字、局部涂抹
   - **41~60**：明显修改，如整个单词或数字被重写、部分遮挡
   - **61~80**：重度修改，如大面积涂抹、覆盖、插入多字、笔画追加
   - **81~100**：严重篡改，原始内容几乎完全被替换或无法辨认
 - **可信度**：高 / 中 / 低
 - **可信程度**：请根据以下标准打分（满分100）：
   - **90~100**：文字非常清晰，笔画完整，无歧义，识别高度可信；
   - **70~89**：文字较清晰，略有模糊或轻微连笔，识别基本可信；
   - **50~69**：文字模糊、部分笔画缺失或存在歧义，识别可信度中等；
   - **30~49**：文字较难辨认，存在多种可能，识别可信度较低；
   - **低于30**：文字极度模糊或无法辨认，识别结果仅供参考；
 - **识别备注**：可注明“原内容为[原始文本]，被手写修改为[当前文本]”、“存在删除线”、“存在涂抹”、“插入文字”、“字体不一致”、“疑似篡改”等。


constraint：
 1. 同一字段只会存在一种字体，要么为印刷体、要么为手写体
 2. 如果同一字段出现多个，并且识别的内容不一致时，请合并识别到的内容
 3. 日期验证：
	3.1 当日期字段完全填写并符合黑白支票图像中手写脚本的指定日期格式时，支票图像应被视为符合日期。否则，将被视为日期不合规。
 4.签名验证
	a. 当验证确认存在一个签名（没有印章）并且符合银行授权的签名规范时，支票图像应被视为符合签名要求。
	b. 当验证确认存在一个印章（没有签名）并且符合银行授权的印章规范时，支票图像应被视为符合印章要求。
	c. 当验证确认确实存在印章和签名（两者）并且符合银行授权的签名和印章规范时，支票图像应被视为符合签名和印章要求。
  当条件a、b和c均未满足时，支票图像应被视为不符合签名或印章。
 5. 重写现有条目以对其进行更改（例如将“3”更改为“8”）;当笔在同一路径上向后移动时，通常是反向移动，以加强或修改原始线条（例如通过收回垂直笔划来加强“K”），就会发生现有的回缩。
 6. 插入字段（在现有文本之间或周围）的额外手写文本会触发文本插入警报。例如，添加最初不存在的数字或单词。
 7. 文本某一部分的字体大小或样式明显不同（例如，一位数字异常大或小得多）表明可能存在篡改。
 8. 穿过文本绘制的水平或对角线（删除线）表示试图取消或删除原始内容。OCR代理将文本标记为可疑。
 9. 完全涂黑或涂抹（使用修正液或浓重笔触）的文本会触发涂黑警报。系统将完全遮挡的内容标记为修改项。
 10. 检测到未经授权添加笔画，将中文大写数字 “一”（对应数值 1）改为 “十”（对应数值 10）。检测到未经授权添加笔画，将英文大写字母 “N” 改为 “M”。
 11. 当支票部分内容的笔迹或墨水颜色与其他部分存在明显差异时，说明该支票可能由多人填写或经过修改。
 12. “Only” 一词之后不得出现任何附加文字。
 13. 修改日期（使用不同墨水 / 笔迹风格或覆盖原始日期）将触发日期篡改警报。

''')

// 2. Get info for KB
n_kb_datas,act_kb.selectBy?sql='SELECT knowledge_name,extension,store_uri FROM kb_spaces_folder_knowledge WHERE id in ({{kbSpaceFileId}})'



// 3. Recognize Cheque by LLM
foreach('kb_data', 'n_kb_datas', {
    // 在循环内部可以访问循环变量
    assign('current_index', '{{kb_data.index}}')
    assign('current_item', '{{kb_data.item}}')
	assign('file_name', '{{current_item.knowledge_name}}.{{current_item.extension}}')
	assign('image_path', '{{current_item.store_uri}}')
    
	// Recognize Cheque by LLM
	n_datas, tpl_image_chat,psr_all

	// Import Data
	n_import_data,act_func?js='''
	(function() {
		let arrList = {{n_datas}};
		for (let i = 0; i < arrList.length; i++) {
			arrList[i].fileName = '{{file_name}}';
			arrList[i].imagePath = '{{image_path}}';
			arrList[i].llm = '{{llm}}';
		}
		return JSON.stringify(arrList);
	})();
	'''

	// Write result into kb
	act_kb.defineAndImportData?tableName='kb_cheque_data'&data='{{n_import_data}}'
})
